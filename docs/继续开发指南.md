# 在其他地方继续开发本工具

代码已托管到 GitHub，与本地机器解耦。任何一台电脑，clone 下来就能改，push 即同步。

## 仓库

- **地址**：https://github.com/chj0124/spec-decision
- **默认分支**：`main`
- **可见性**：Public（如需私有，到仓库 Settings → Danger Zone → Change visibility）

---

## 一、在新电脑上开始

```bash
# 1. 克隆
git clone https://github.com/chj0124/spec-decision.git
cd spec-decision

# 2. 配置 git 身份（首次）
git config user.name "chj0124"
git config user.email "chj0124@users.noreply.github.com"

# 3. 国内环境配代理（GitHub 被墙时）
git config http.proxy http://127.0.0.1:7897
git config https.proxy http://127.0.0.1:7897

# 4. 装依赖（国内用淘宝镜像更快）
npm install --registry=https://registry.npmmirror.com

# 5. 起开发服务器
npm run dev        # → http://localhost:5173
```

需要 Node 18+（推荐 22）。没装的话先去 https://nodejs.org 装 LTS 版。

---

## 二、日常开发流程

```bash
# 改代码...
npm run dev          # 边改边看

# 提交前验证
npm run build        # tsc 类型检查 + vite 构建，必须通过

# 提交 + 推送
git add -A
git commit -m "feat: 描述这次改动"
git push
```

**铁律**：`git push` 前一定先 `npm run build` 跑通，避免把编译不过的代码推上去。

## 三、Vercel 自动部署（配一次，以后 push 即上线）

只需在 Vercel 网页关联一次，之后每次 `git push` 自动构建发布：

1. 打开 https://vercel.com ，用 GitHub 账号登录
2. **Add New → Project** → 选 `chj0124/spec-decision`
3. Vercel 会自动识别 **Vite** 框架：
   - Build Command：`npm run build`（默认，不用改）
   - Output Directory：`dist`（默认，不用改）
   - 仓库里已有 `vercel.json`，会自动生效
4. 点 **Deploy**，约 1 分钟出正式链接（形如 `spec-decision.vercel.app`）

配好后：
- `git push` → Vercel 自动拉代码 → 构建 → 上线
- 每次 push 生成一个预览链接，`main` 分支对应正式域名

### 环境变量（如需 AI 截图识别走视觉端点）

在 Vercel 项目 **Settings → Environment Variables** 加：

| 变量 | 用途 |
|------|------|
| `DASHSCOPE_API_KEY` | 通义千问视觉模型密钥（Serverless 识别端点用） |
| `RECOGNIZE_PROVIDER` | 识别服务商，默认 `qwen` |

> 用户在网页里配的 AI（单位换算/文案润色）走的是浏览器端配置，不依赖 Vercel 环境变量。

## 五、常见问题

**push 时卡在授权 / 403**
→ 本机 Git Credential Manager 已存凭据可直推；新电脑首次 push 会弹 GitHub 授权窗口，登录即可。国内务必先配代理（见上）。

**npm install 慢 / 失败**
→ 加淘宝镜像 `--registry=https://registry.npmmirror.com`。

**构建报单位/类型错**
→ 先 `npm run build` 看完整报错；本项目单位归一化逻辑在 `src/lib/engine.ts`，双向同步在 Workbench/RecognizeReview。

**想改但怕搞乱**
→ 开个分支 `git checkout -b my-feature`，改完合并回 `main`；或先 push 到新分支，在 GitHub 上开 PR 对比。
